# yandex_vacancy_search

Автоматизированный поиск вакансий на Яндекс.Работе + фильтрация через LLM под ваш стек.

> Если первичный найм автоматизирован, то и поиск вакансий можно автоматизировать подумал я. Подумал, можно и сделать.

## Что делает конкретно (пайплайн)

1. Парсит вакансии с Яндекса (через внутренний API, не HTML а значит есть пагинация)
2. Хранит их в SQLite (без дублей)
3. Удаляет закрытые вакансии (то на что вы уже откликнулись или на что вам уже отказали п.4)
4. Учитывает blacklist (вакансии, куда уже откликнулся / отказали)
5. Прогоняет новые вакансии через LLM (API OpenRouter. Он копеечный и за пару баксов может жить месяц)
6. Формирует отчет с релевантными вакансиями

---

## Как конкретно работает

* уникальность по `url`
* обновление `matched_keywords` (если вакансия попалась по разным запросам. А в смежных отраслях типо DS/ML такое часто бывает)
* LLM обрабатывает только новые вакансии (не переживайте за ваши токены опенроутер)
* отчет содержит только релевантные вакансии (`score >= threshold`. Но учтите что решение о релевантности принимает LLM, а не вы сами)

## Запустить очень просто

```bash
git clone ...
cd project

python -m venv venv
source venv/bin/activate  # или venv\Scripts\activate на Windows

pip install -r requirements.txt
```

Настройка:

* `.env` (зарегайтесь на OpenRouter и добавьте ключ. Используйте .env.example для примера)
* `profile.txt` (опишите свой стек и интересы. Это по сути промт для LLM-ки. Она должна знать кто вы)
* `system_prompt.txt` (ну и сам системный промт как вас оценивать)

Запустите:

```bash
python app.py
```

Откройте в браузере на пк:

```
http://127.0.0.1:5000
```

## Использование

В UI три кнопки:

### 1. Парсер

* собирает вакансии
* добавляет новые
* удаляет закрытые
* применяет blacklist

### 2. LLM

* обрабатывает только новые вакансии
* оценивает релевантность под ваш профиль

### 3. Отчет

* формирует файл в `/reports`
* содержит только релевантные вакансии
